# Tech Product Blog Writer

This is the Codex-readable mirror for `.cursor/skills/tech-product-blog-writer/SKILL.md`.
Read the Cursor skill for the full rule set before drafting a tech product review.

## Review Pattern

- Open with the target user and practical use case.
- Lead with concrete strengths and real usage outcomes.
- Use specific product attributes: build, ergonomics, portability, battery/charging, display, connection, remote, tripod stability, height range, compatibility.
- Compare briefly with same-segment alternatives when useful.
- If price is mentioned, label it as a timestamped reference and note that marketplace promotions, vouchers, and shop campaigns can change the final checkout price.
- Keep limitations short, balanced, and tied to practical trade-offs.
- End with a clear value judgment: who the product fits and why.

## Product Review Must-Haves

- Bold key highlights for scanability.
- Avoid unsupported claims and over-promising.
- Cover banner must click through to the primary affiliate URL.
- Source links must point to the product listing and relevant marketplace/source pages.
